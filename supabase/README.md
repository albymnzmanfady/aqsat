# نشر التطبيق على VPS

## كيف تعمل قاعدة البيانات؟

البيانات محفوظة في **Supabase Cloud** وليس في الكود.

### عند النسخ من GitHub إلى VPS:

1. **لا حاجة لإعداد قاعدة بيانات** - يعمل تلقائياً
2. الملف `src/integrations/supabase/client.ts` يحتوي على:
   - رابط Supabase
   - مفتاح API

### خطوات النشر على VPS:

```bash
# 1. نسخ الكود
git clone your-repo.git
cd your-repo

# 2. تثبيت المكتبات
npm install

# 3. بناء التطبيق
npm run build

# 4. تشغيل الخادم
npm start
```

### النسخ الاحتياطي:

- **البيانات دائماً في Supabase** - لا تتأثر بـ VPS
- استخدم صفحة الإعدادات → النسخ الاحتياطي للتصدير/الاستيراد
- يمكنك تصدير JSON في أي وقت للاحتفاظ بنسخة محلية

### هل أحتاج لإعادة إعداد المستخدمين؟

**لا!** المستخدمون والبيانات كلها في Supabase.
فقط تأكد من أن ملف `src/integrations/supabase/client.ts` موجود ومحدث.

## ملاحظة أمنية مهمة:

ملف `server/supabase.ts` على الخادم يحتوي على نفس المفتاح -
**لا تقم بتشغيل الخادم المحلي مع Supabase** إلا إذا كنت تحتاجه.

التطبيق يعمل الآن مباشرة مع Supabase من المتصفح
بدون حاجة للخادم المحلي.
